Hello, and you are very welcome.

The Audio SDK interface is rather tricky for the MJPEG cameras, when not using IE ("Internet Explorer") or ActiveX.

This is because it requires the use of TCP/IP packets to send and receive audio buffers/data to and from the camera.

Once you get the audio buffers ("The buffers are in ADPCM .wav format, without any .wav headers"), you will need to create your own methods to send those buffers to a speaker on the computer or device, to be heard/played.

The same goes with needing access to the microphone of the computer or device to get the audio buffers to send to the camera.

This low level TCP/IP interface also allows real-time access to the alarm status messages from the cameras and video streaming as well as to logon and other things, which can also, be done at a low level.

In fact, the ActiveX IE Browser interface is using this low level TCP/IP interface combined with the .cgi interface, when you use the standard camera management interface via IE with MJPEG based cameras.

There is very little documentation, publicly available, about this low level TCP/IP interface, for the MJPEG cameras.

Here is some documentation on the this low level interface translated from Chinese to English. Please note that this interface, does not apply to the H.264 cameras, which uses different interface methods:

http://translate.googleusercontent.com/translate_c?hl=en&prev=/search%3Fq%3DIP%2Bcamera%2B%2522MO_V%2522%2Bcommand%26hl%3Den%26safe%3Doff%26biw%3D1366%26bih%3D599%26prmd%3Dimvns&rurl=translate.google.com&sl=zh-CN&u=http://www.sistemasorp.es/blog/IPCameraProtocol.doc&usg=ALkJrhh5Myjqd7jY1IeW24FWC4dx8LeFXA

In your case, you may wish to use the original document and have Google translate it from Chinese to Korean. The original document in Chinese, can be found here:

http://www.sistemasorp.es/blog/IPCameraProtocol.doc

Addtionally, there is a more extensive version of the above, in a .zip file that is in Chinese and includes how to do camera movements and controls using the low level interface, which can be found here:

http://www.vworker.com/RentACoder/DotNet/misc/DownloadFile.aspx?lngDownloadTypeId=2&lngInteger1=1601042&txtForceRefresh=0000

Once you unzip the above, you can select all the text in the .pdf, and use Google Translate to translate to English or Korean or any other language, using a paste of the copied .pdf text.

If your camera is in wireless mode, while you are testing this interface, you can use WireShark, to see the TCP/IP packet flow from the camera to a computer or device.

If you do not yet have WireShark you can get it here, it is free:

http://www.wireshark.org/

Before even starting any programming, you can trap these low level TCP/IP packets, which are sent even when using the normal camera interface.

For best results, you may wish to use a IE ("Browser") and access the standard camera management system.

You will be able to start/stop audio using the standard camera management interface ("IpAddress:Port") and see these low level packets when you do. You will be able to see logon, alarms, video streaming and audio packets this way using WireShark.

Note: These low level TCP/IP packets show up labeled in your WireShark traces as "Continuation or Non-HTTP traffic", so they are easy to find and locate in the mix of the other TCP/IP traffic, going on at the same moment in time.

by dougiboy » Tue Dec 25, 2012 7:18 am

It would seem the Foscam IP Camera has the capability to be set up with an External Speaker and Microphone. However, this is not straight forward. 1) First the 3.5mm Plugs, Length TBD - we must assume they are standard mono plugs. With a break contact to turn off the internal speaker. On a 8910 - the 3.5mm speaker plug has to be pulled back about 1mm in order for the external speaker (Lansing Speaker Impedance to confirm) to function and turn off the internal speaker. 2) Micro Phone - will be a Mono Microphone with a 3.5mm plug - still working on. 3) Have seen on this Forum issues with noise - one comment suggests hardwire to router rather than Wireless may help. Suggest a list of tried and tested Speakers and Mic's would be good.
